## 3. Проблема

| Вопрос | Ответ |
|--------|-------|
| **Какую проблему решаете?** | Поиск информации в тысячах промышленных документов (ГОСТ, СТО, чертежи) занимает часы |
| **Кто сталкивается?** | Инженеры, технологи, нормоконтролёры (50+ человек) |
| **Как справляются сейчас?** | Ручной поиск в папках, Excel-таблицы, запросы в архив |
| **Масштаб** | 10,000+ документов, 50+ запросов/день, 2-3 часа на поиск одного документа |

---

## 4. Решение

**Что предлагаете?**
Локальная система на базе Ollama + ChromaDB + Agno, которая индексирует все документы и отвечает на вопросы на русском языке с указанием источников.

**Роль LLM:**
- **Ядро системы** — суммаризация, извлечение сущностей, поиск по смыслу
- **Почему LLM?** — понимание контекста, работа с неструктурированным текстом, русский язык

**Технический подход:**
| Компонент | Технология |
|-----------|------------|
| OCR | MinerU + vLLM (qwen3-vl) |
| NLP | spaCy (ru_core_news_lg) |
| LLM | Ollama (qwen3:14b, qwen2.5:14b) |
| Vector DB | ChromaDB Server |
| Agents | Agno Framework |
| Паттерн | RAG + Metadata-aware search |

**Пользовательский сценарий:**
```
1. Пользователь: "Что сказано в ГОСТ 12.1.004-91 про опасные факторы пожара?"
2. Система: Поиск в ChromaDB → Rerank по метаданным → Генерация ответа через Agno
3. Результат: Краткий ответ + 3-5 ключевых пунктов + источники (ГОСТ, дата, файл)
```
